# Codex 技能使用建议（当前项目精简版）

这份清单只服务当前项目主线：

- 单目视觉
- 海参检测
- 双关节六绳驱动软体机器人
- 电脑通过 USB/TTL 直控六个舵机
- 论文写作、实验整理与图表输出

当前仓库里没有实际落地的 `.agents/skills/` 目录，因此这里不再扩展一大批自定义技能模板，只保留对当前阶段真正有用的技能建议。

---

## 1. 建议常驻保留的技能

这些技能与当前项目高度相关，建议保留在活动集合中：

- `0-autoresearch-skill`
  - 用于维持研究工作区、阶段总结和问题追踪。
- `doc`
  - 用于读写论文草稿、开题报告、任务书和答辩文稿。
- `pdf`
  - 用于审阅本地文献、检查论文 PDF 排版、查看图表与公式布局。
- `spreadsheet`
  - 用于实验结果表、误差统计表、参数记录和结果汇总。
- `academic-plotting`
  - 用于生成论文里的结构图、对比图和实验图表。
- `ml-paper-writing`
  - 用于把研究内容整理成论文式表达。
- `systems-paper-writing`
  - 用于补强系统实现、实验设计和结果分析的结构化写法。
- `presenting-conference-talks`
  - 用于整理答辩稿、演示提纲和逐页讲稿。
- `imagegen`
  - 用于快速生成示意图素材、结构草图和答辩辅助配图。

系统层技能也应保留：

- `.system`
- `codex-primary-runtime`

---

## 2. 当前不建议常驻使用的技能类型

以下技能类别与当前阶段关系较弱，不适合作为活动集合的默认常驻项：

- 大模型训练与微调框架
  - 如 `axolotl`、`trl-fine-tuning`、`unsloth`、`deepspeed`
- 模型量化与压缩
  - 如 `awq`、`gptq`、`bitsandbytes`、`gguf`
- 大规模评测与推理服务
  - 如 `lm-evaluation-harness`、`vllm`、`sglang`
- 云平台与分布式训练
  - 如 `modal`、`lambda-labs`、`ray-train`、`skypilot`
- 与当前项目无直接关系的多模态或前沿研究工具
  - 如大部分通用 LLM 训练、向量数据库、远程解释与高性能并行类技能

这类技能不是“永远不用”，而是对当前毕业设计主线贡献较低，默认不必占用活动集合。

---

## 3. 当前阶段最值得直接调用的工作流

如果只保留最实用的调用方式，当前阶段建议优先这样使用：

- 用 `0-autoresearch-skill` 维护研究状态、问题清单和阶段总结
- 用 `doc` / `pdf` 处理论文草稿和本地文献
- 用 `spreadsheet` 统计实验误差、对准时间和结果表
- 用 `academic-plotting` 画结构图、控制流程图和实验结果图
- 用 `ml-paper-writing` 或 `systems-paper-writing` 把工程实现整理成论文章节
- 用 `presenting-conference-talks` 压缩答辩材料

---

## 4. 当前不再推荐扩展的旧方向

以下方向在当前阶段不再作为主线技能规划重点：

- `ros2-stm32-bridge` 这类围绕 ROS2 + STM32 分层架构的自定义技能设想
- 过度细分的大量论文写作技能模板
- 与当前代码和实验路径脱节的大模型研究型技能矩阵

`src/Gradua` 中的 ROS 包保留为后续改进储备，但不需要继续围绕它扩展技能体系。

---

## 5. 使用原则

1. 当前优先保留少量高频、直接有用的技能，而不是追求技能数量。
2. 任何技能如果不能直接帮助当前这条主线：
   - 视觉误差提取
   - 六舵机控制
   - 软体机器人实验
   - 论文与答辩输出
   就不建议常驻保留。
3. 如果后续项目重新转回 ROS 化、分层控制或更复杂的软件栈，再按新需求补回对应技能。
