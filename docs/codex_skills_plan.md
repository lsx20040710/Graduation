# Codex 毕设写作与工程实现技能清单

这份清单配套上面的 `AGENTS.md` 使用。思路是：

- `AGENTS.md` 放长期、稳定、每次都要遵守的规则
- `.agents/skills/` 放按任务触发的“专门工作流”
- 这样更符合 Codex 官方推荐的“AGENTS 管长期规则，Skills 管可复用流程”的分工

---

## 一、先装的官方技能

### 1) 内置系统技能
这两个一般是现成就有的：

- `$skill-creator`
- `$skill-installer`

### 2) 推荐安装的官方通用技能

```bash
$skill-installer doc
$skill-installer pdf
$skill-installer spreadsheet
```

按需再装：

```bash
$skill-installer openai-docs
$skill-installer imagegen
```

### 3) 这些技能适合你的原因

- `doc`：适合读写 `.docx`，特别适合论文正文、开题报告、任务书、答辩稿
- `pdf`：适合审阅论文 PDF、检查排版、看公式和图表布局
- `spreadsheet`：适合实验数据、误差统计、对比表格、指标汇总
- `openai-docs`：只有在你真的在项目里用 OpenAI API / Codex / MCP 时才值得装
- `imagegen`：用于画流程图草图、示意图素材、论文配图草稿

## 1. thesis-source-scout

### 用途
从零开始做文献检索、筛选、归类，形成“可写综述”的文献池。

### 触发场景
- “帮我找近五年视觉伺服相关文献”
- “帮我整理 IBVS、PBVS、HVS 的代表论文”
- “给我一组能支撑第二章综述的文献”

### 它应该做的事
1. 先拆主题词
2. 先给检索式
3. 再给候选文献
4. 再按主题归类
5. 最后输出“可直接写进文献综述”的整理结果

### 输出格式
- 研究主题
- 检索关键词
- 核心文献
- 每篇文献的：问题、方法、结论、局限、可引用位置
- 综述写法建议

---

## 2. thesis-outline-builder

### 用途
快速把一个课题变成“像论文”的大纲和章节逻辑。

### 触发场景
- “帮我搭论文提纲”
- “帮我做目录”
- “帮我按本科毕设格式拆章节”

### 它应该做的事
1. 根据课题类型判断是系统设计型、控制算法型、实验验证型还是综合型
2. 生成 5~6 章结构
3. 每章拆到二级/三级标题
4. 标出每节要写什么、需要什么图表、需要哪些证据

## 3. chapter-drafter-zh

### 用途
写中文工科论文章节初稿，尽量减少 AI 味。

### 触发场景
- “写 3.2 节”
- “把这段改成论文风格”
- “帮我补完整个第四章初稿”

### 它应该做的事
1. 先给本节目标和小节结构
2. 再给正文
3. 默认句式不要太整齐
4. 默认保留引用占位符 `[R1]`
5. 不编造实验结果和引用

### 风格要求
- 中文工科论文风
- 少空话
- 少机械连接词
- 术语统一
- 能直接进入后期修改

---

## 4. citation-audit

### 用途
专门查“引用是否像真的、是否缺证据、是否有伪引风险”。

### 触发场景
- “检查这段有没有需要补引用的地方”
- “帮我审一遍参考文献风险”
- “哪些句子容易被老师问来源”

### 它应该做的事
1. 标出事实性陈述
2. 标出必须补引文的位置
3. 标出高风险句子
4. 标出看起来像 AI 编的引用
5. 给出“缺证据清单”

---

## 5. ros2-stm32-bridge

### 用途
服务你的实际工程：ROS2 上位机与 STM32 下位机协同。

### 触发场景
- “帮我设计 ROS2 + STM32 的通信链路”
- “帮我解释节点架构”
- “帮我写系统软件架构一节”

### 它应该做的事
1. 列清楚节点、话题、消息类型、频率
2. 解释上位机与下位机分工
3. 帮你把工程实现转成论文表达
4. 给出架构图建议

---

## 6. visual-servo-modeling

### 用途
专门服务视觉伺服建模与控制章节。

### 触发场景
- “IBVS 和 PBVS 怎么写到论文里”
- “帮我写视觉雅可比”
- “帮我解释视觉-运动耦合模型”

### 它应该做的事
1. 区分图像空间、相机空间、执行器空间
2. 定义误差变量
3. 解释控制输入和输出
4. 给出适合本科论文的推导深度
5. 优先保证可解释，不盲目堆公式

---

## 7. experiment-report-packager

### 用途
把实验记录整理成“论文实验章节”。

### 触发场景
- “帮我把实验数据写成 5.3 节”
- “把这组结果整理成实验分析”
- “帮我做静态对准和动态跟踪对比”

### 它应该做的事
1. 先整理实验目的
2. 再列平台和参数
3. 再写结果
4. 再写分析
5. 强制区分“现象描述”和“机理解释”

---

## 8. defense-materials-cn

### 用途
答辩前一键产出：摘要、PPT 文案、讲稿、问答。

### 触发场景
- “帮我做答辩讲稿”
- “老师可能问什么”
- “把论文压缩成 5 分钟口述版”

### 它应该做的事
1. 抽取论文主线
2. 生成答辩摘要
3. 生成逐页讲稿
4. 生成高频问答
5. 帮你统一口径，避免前后矛盾

### A. thesis-source-scout/SKILL.md

```md
---
name: thesis-source-scout
description: 用于本科毕业设计的文献检索、筛选、归类与综述前整理。仅在用户要求找文献、做综述、比较方法、整理研究现状时触发；不用于直接生成整篇论文正文。
---

# Thesis Source Scout

## 目标
帮助用户从零构建可信文献池，优先输出“可用于写综述”的结构化结果，而不是零散文献列表。

## 强制规则
1. 优先高质量来源：Google Scholar 可检索论文、IEEE Xplore、Springer、Elsevier、Wiley、学位论文、标准文档、官方文档。
2. 不把 CSDN、知乎、博客园当作论文理论依据。
3. 不编造参考文献、DOI、页码、卷期。
4. 如果来源信息不完整，明确标记“待核实”。

## 工作流
1. 识别研究主题与子主题。
2. 为每个子主题生成检索关键词（中文+英文）。
3. 给出检索式建议。
4. 输出候选文献并按主题分类。
5. 对每篇文献提炼：
   - 研究问题
   - 方法
   - 主要结果
   - 优点
   - 局限
   - 与用户课题的关系
6. 最后给出：
   - 综述写作逻辑
   - 推荐引用位置
   - 当前证据缺口

## 默认输出格式
### 主题
### 检索关键词
### 推荐文献
### 综述归类
### 本课题可切入的问题
```

```md
name: chapter-drafter-zh
description: 用于撰写中文工科本科毕业设计章节初稿。仅在用户要求写章节、扩写提纲、改成论文风格、降 AI 味时触发；不用于伪造实验结果或伪造引用。

# Chapter Drafter ZH

## 目标
生成可直接进入论文修改流程的中文工科章节初稿，强调结构完整、术语统一、语言自然。

## 强制规则
1. 先给小节结构，再给正文。
2. 每段围绕一个核心论点展开。
3. 不伪造实验结果、图号、表号、公式编号、引用。
4. 需要引用的位置用 [R1] [R2] 占位。
5. 少用空泛表述，避免整段“首先其次然后最后”堆砌。

## 默认写法
1. 写本节目标
2. 写本节结构
3. 写正文
4. 补充本节需要的图表/公式/实验依据

## 风格
- 中文
- 工科论文
- 克制、准确、具体
- 能通过人工继续精修
```

### C. citation-audit/SKILL.md

```md
---
name: citation-audit
description: 用于检查论文草稿中的事实性陈述、引用缺口、伪引风险与高风险表述。仅在用户要求检查参考文献、补引用、审查引用真实性时触发。
---

# Citation Audit

## 目标
帮助用户快速发现论文中最容易在检查时暴露的问题：没来源、来源弱、像编的、结论过度。

## 工作流
1. 通读文本，标出事实性陈述和结论性判断。
2. 判断哪些句子必须有引用。
3. 标出高风险表述：
   - 明确数据但无出处
   - “大量研究表明”但无代表文献
   - “显著提高/明显优于”但无实验支撑
   - 年份、机构、作者信息可疑
4. 输出缺证据清单。
5. 输出修改建议：
   - 该补哪类文献
   - 该弱化哪些语气
   - 该删掉哪些高风险句子

## 输出格式
### 必须补引文的位置
### 可能存在伪引风险的位置
### 可弱化表述的位置
### 推荐补充的证据类型
```
